The Initiate phase identifies the purpose for the claim, and the intended participants (client, employee, claim provider).<br /></p><p><br /></p><p>Dialog is captured against the client (CustomerInteraction), and SLA are captured and initialised.<br /></p><p><br /></p><p>We have a number of implementation options, each with pro's and cons.<br /></p><p><ol><li>UI Screen to enter CRN/Product ID/Customer ID.  ID is verified & basic information populated before continuing.<br />Pros: Comms/Activity created automatically;  Richer UI to lookup details - therefore a better user experience<br />Cons: Slightly more effort in development for UI - expectation management for richer UI screens</li><br /></p><p><li>UI Screen to enter CRN/Product ID/Customer ID.  Process moves onto next step to retrieve information, and user gets a task to verify that the entered details are correct.<br />Similar to #1, but less effort, reduced user experience.  One screen to initiate, then a task to verify.  Takes longer in the case of incorrect information being entered.</li><br /></p><p><li>Automatic initiation of claim triggered by detecting the Comms.  <br />Pro: Guarantees that the correct product/client is chosen, simplified UI<br />Con: Manually created comms / activity with possibly inconsistent information</li><br /></p><p></ol><br /></p>
