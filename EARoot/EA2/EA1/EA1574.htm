Confirm forms required.<br /></p><p><br /></p><p>Review and confirm basic recipient name and contact details, and update/confirm delivery method.<br /></p><p><br /></p><p>A review interface allows the participant to check and include/exclude forms that are not required.  Interface should be aware that this process can loop, and forms that have already been submitted should be correspondingly indicated.<br /></p><p><br /></p><p>Each form will indicate who is responsible for it's completion.  This can be overridden, but only to re-allocate responsibility internally (i.e. from the GP to STL).<br /></p><p><br /></p><p>The Pension consultant reviews and confirms basic Client details, as well as the target address and method of correspondence for each form recipient (excluding Centrelink).  <br /></p><p><br /></p><p>Show the last modified date of client information.<br /></p><p><br /></p><p>Participant can also override delivery method (electronic / physical).<br /></p><p><br /></p><p>Participant can choose to restart this process to pick up any core system changes that may have resulted from the review.<br /></p><p><br /></p><p><br /></p><p>Optional (stretch goal):<br /></p><p><br /></p><p>Update process instance specific data that cannot be obtained from core system and must not be left blank.<br /></p><p><br /></p><p>Stretch Goal - Ability to update claim specific data that cannot be obtained from source systems.  Updates should be audited - particularly once the claim has been submitted for the first time.<br /></p>
