A new load will often require a new extract from a source system.  The extract is constructed by defining an extract service on the ESB that will pull data from the application system and load it into a document for later pick-up by one or more load services.<br /></p><p><br /></p><p>The extract service on the ESB will call an application component that will pull data from the application and load it into a document (usually in the canonical data format) where it can later be picked up by receiving applications.<br /></p><p><br /></p><p>When an extract job is complete the application component will return a message to the ESB indicating that the data update is complete. <br /></p><p><br /></p><p>In the case where bulk data updates are created this message will indicate the location where the document containing the updates was written. <br /></p><p><br /></p><p>Where only a few update records are created these updates will be contained in the message itself.<br /></p><p><br /></p><p>Applications that need to apply the updates to their own internal data stores will subscribe to the ESB - listening for update messages.  When an update message is detected it will invoke its own application component, translate the update data into a form consistent with the application and then load it. As with the extract process it is preferable if this load process uses some form of API or web services to effect this load but the data can be loaded directly into the database if necessary.<br /></p><p>The batch manager will schedule the extract and / or load service sequences as necessary.  In some cases these sequences will be triggered by a message coming from the clock, in some cases by business events and in other cases the trigger will be by external or manual events. If the sequence is triggered by external of manual events and must occur with a set frequency the Batch Manager will ensure that this frequency is met.  If it is not met then emails will be sent and the issue escalated until a trigger is received so that the sequence can proceed.<br /></p><p><br /></p><p>Errors that are detected by the extract and load services will, as with other service errors, be forwarded to an error management service. <br /></p>
