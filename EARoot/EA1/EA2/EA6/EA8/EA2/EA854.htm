This component allows source system records (SSR) to be matched.  <br /></p><p><br /></p><p>Data Matching is the ability to provide probabilistic matching to any relevant attribute – evaluating user-defined full fields, parts of fields, or individual characters.  This is done by assigning agreement weights and disagreement weights to key data elements, based on a number of factors such as frequency distribution, discriminating value, and reliability. <br /></p><p><br /></p><p>Data Matching results can be used for linkages and de-duplication. <br /></p><p><br /></p><p>For Master data, an algorithm that calculates the probability that the records are in-fact representing the same underlying business object based on configuration data (such as attribute matching conditions) is used to automatically to detect if an SSR matches an existing ODS record.  If a match is detected the SSR  can be automatically linked to the ODS record and all other links to that ODS record will show a linkage between various source system records.  Likely matches are also detected and these can be forwarded to a workflow queue for manual decision making.<br /></p><p><br /></p><p>The Master Data service will synchronise data from multiple systems that may contain overlapping data.   It will keep a record of source system data as required and will also keep a central record of the master data.  This design requires more data storage but increases the flexibility and extensibility of the system.  <br /></p><p><br /></p><p>The ODS will be used to store the centrally held master records and respond to requests for master data.<br /></p>
