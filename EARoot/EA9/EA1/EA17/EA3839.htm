<b>Description</b><br /></p><p><br /></p><p>A task in a process in enabled after the completion of a preceding task in the same process.<br /></p><p><br /></p><p><b>Synonyms</b><br /></p><p><br /></p><p>Sequential routing, serial routing.<br /></p><p><br /></p><p><b>Motivation</b><br /></p><p><br /></p><p>The Sequence pattern serves as the fundamental building block for processes. It is used to construct a series of consecutive tasks which execute in turn one after the other. Two tasks form part of a Sequence if there is a control-flow edge from one of them to the next which has no guards or conditions associated with it.<br /></p><p><br /></p><p><b>Context</b><br /></p><p><br /></p><p>There is one context condition associated with this pattern: an instance of the Sequence pattern cannot be started again until it has completed execution of the preceding thread of control (i.e. all places such as p1 in the Sequence must be safe).<br /></p>
